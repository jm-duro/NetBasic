_machine_

-- Machine Level Access

procedure

Following global procedures are defined:

  • poke8 (atom, object)

    Params:
    • atom address

      either a long integer or a non empty sequence of long integers

    • object x

    Example:
    poke8(addr, -1)

    See also:

    peek8u(), peek8s()

  • poke_float (atom, atom)

    Params:
    • atom p

    • atom a

    Example:
    poke_float(addr, 3.1416)

    See also:

    peek_float()

  • poke_double (atom, atom)

    Params:
    • atom p

    • atom a

    Example:
    poke_double(addr, 216656332.2)

    See also:

    peek_double()

  • poke_pointer (atom, object)

    Params:
    • atom addr

    • object x

    Example:
    poke_pointer(addr, allocate_string("where?"))

    See also:

    peek_pointer()

  • poke_plist (atom, sequence)

    Params:
    • atom addr

    • sequence pointers

    Example:
    addr2 = allocate(4)
    addr3 = allocate(4)
    addr = allocate_plist({NULL, NULL})
    poke_plist(addr, {addr3, addr2})

    See also:

    allocate_plist(), peek_plist(), free_plist()

  • free_plist (atom)

    Params:
    • atom addr

    Example:
    free_plist(addr)
    ? peek_plist(addr) --> {}

    See also:

    allocate_plist(), poke_plist(), peek_plist()

  • free_string_list (atom)

    Params:
    • atom addr

    Example:
    free_string_list(addr)
    s = peek_string_list(addr, {})
    s is {}

    See also:

    allocate_string_list(), peek_string_list()

function

Following global functions are defined:

  • peek8u (object)

    Params:
    • object address

      either of
      * an atom addr : to fetch one long integer at addr, or
      * a pair {addr,len} : to fetch len long integers at addr

    Example:
    poke8(addr, -1)
    ? peek8u(addr) --> 1.844674407e+019

    See also:

    poke8(), peek8s()

  • peek8s (object)

    Params:
    • object address

      either of
      * an atom addr : to fetch one long integer at addr, or
      * a pair {addr,len} : to fetch len long integers at addr

    Example:
    poke8(addr, -1)
    ? peek8s(addr) --> -1

    See also:

    poke8(), peek8u()

  • peek_float (atom)

    Params:
    • atom p

    Example:
    poke_float(addr, 3.1416)
    ? peek_float(addr) --> 3.141599894 (caused by float storage format)

    See also:

    poke_float()

  • peek_double (atom)

    Params:
    • atom p

    Example:
    poke_double(addr, 216656332.2)
    ? peek_double(addr) --> 216656332.2

    See also:

    poke_double()

  • peek_pointer (object)

    Params:
    • object addr

    Example:
    addr2 = allocate_string("where?")
    ? addr2
    poke_pointer(addr, addr2)
    ? peek_pointer(addr) --> same as addr2

    See also:

    poke_pointer()

  • allocate_plist (sequence)

    Params:
    • sequence pointers

    Example:
    addr2 = allocate(4)
    addr3 = allocate(4)
    addr = allocate_plist({addr2, addr3})

    See also:

    poke_plist(), peek_plist(), free_plist()

  • peek_plist (atom)

    Params:
    • atom addr

    Example:
    addr2 = allocate(4)
    ? addr2 --> 4758384
    addr3 = allocate(4)
    ? addr3 --> 4832984
    addr = allocate_plist({addr2, addr3})
    ? peek_plist(addr) --> {4758384,4832984}

    See also:

    allocate_plist(), poke_plist(), free_plist()

  • allocate_string_list (object)

    Params:
    • object string_list

    Description:

    no need of poke_string_list as allocate_string_list does the job

    Example:
    addr = allocate_string_list({"my","taylor","is","rich"})

    See also:

    peek_string_list(), free_string_list()

  • peek_string_list (atom, integer)

    Params:
    • atom addr

    • integer max

      fixes maximum number of strings to read

    Description:

    opposite of allocate_string_list()

    Example:
    addr = allocate_string_list({"my","taylor","is","rich"})
    s = peek_string_list(addr)
    s is {"my","taylor","is","rich"}
    s = peek_string_list(addr, 2)
    s is {"my","taylor"}

    See also:

    allocate_string_list(), free_string_list()

  • cdecl_callback (integer)

    Params:
    • integer rtn

  • byte_to_word (atom, atom)

    Params:
    • atom l

    • atom h

    Example:
    ? byte_to_word(#AA, #80) --> 32938 (#80AA)

    See also:

    high_byte(), low_byte()

  • high_byte (atom)

    Params:
    • atom x

    Example:
    ? high_byte(#80AA) --> 128 (#80)

    See also:

    byte_to_word(), low_byte()

  • low_byte (atom)

    Params:
    • atom x

    Example:
    ? low_byte(#80AA) --> 170 (#AA)

    See also:

    byte_to_word(), high_byte()

  • word_to_dword (atom, atom)

    Params:
    • atom l

    • atom h

    Example:
    ? word_to_dword(#21FA, #8123) --> 2166563322 (#812321FA)

    See also:

    high_word(), low_word()

  • high_word (atom)

    Params:
    • atom pData

    Description:

    from win32lib tk_maths.e

    Example:
    ? high_word(#812321FA) --> 33059 (#8123)

    See also:

    word_to_dword(), low_word()

  • low_word (atom)

    Params:
    • atom pData

    Description:

    from win32lib tk_maths.e

    Example:
    ? low_word(#812321FA) --> 8698 (#21FA)

    See also:

    word_to_dword(), high_word()

  • higher_bits (integer, integer)

    Params:
    • integer x

    • integer divide

      divide is the is the value that fixes the limit between higher bits and lower bits

    Example:
    higher_bits(x, #10) gets the higher quartet (4 bits) of byte x (8 bits)
    higher_bits(x, #100) gets the higher bytes (8 bits) of word x (16 bits)
    higher_bits(x, #10000) gets the higher word (16 bits) of dword x (32 bits)

  • lower_bits (integer, integer)

    Params:
    • integer x

    • integer divide

      divide is the is the value that fixes the limit between higher bits and lower bits

    Example:
    lower_bits(x, #10) gets the lower quartet (4 bits) of byte x (8 bits)
    lower_bits(x, #100) gets the lower bytes (8 bits) of word x (16 bits)
    lower_bits(x, #10000) gets the lower word (16 bits) of dword x (32 bits)

  • join_bits (integer, integer, integer)

    Params:
    • integer highest

    • integer lowest

    • integer divide

      divide is the resulting value when highest is 1 and lowest is 0

    Example:
    join_bits(h, l, #10) joins to quartets (4 bits) to build a byte (8 bits)
    join_bits(h, l, #100) joins to bytes (8 bits) to build a word (16 bits)
    join_bits(h, l, #10000) joins to words (16 bits) to build a dword (32 bits)
